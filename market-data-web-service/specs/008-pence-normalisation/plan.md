# Implementation Plan: Pence Unit Normalisation

**Branch**: `008-pence-normalisation` | **Date**: 2026-05-26 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/008-pence-normalisation/spec.md`

## Summary

Time series quoted in sub-unit currencies (e.g. GBp вЂ” British pence, USd вЂ” US cents) must be automatically scaled to their major-unit equivalent (Г·100) and reported with the corresponding major-unit currency code (GBP, USD) across all pricing endpoints. A new `SubUnitNormaliser` service class is injected into `PricingService`, where it normalises both the price value and the currency string after data is retrieved from the provider chain and before the response models are constructed. This ensures FX translation (an optional downstream step) always operates on correctly-scaled major-unit values.

## Technical Context

**Language/Version**: Python 3.12  
**Primary Dependencies**: FastAPI, pydantic v2, yfinance, structlog, pytest, pytest-bdd  
**Storage**: Parquet file cache (local disk) вЂ” untouched by this feature  
**Testing**: pytest + pytest-bdd (BDD-first, Gherkin `.feature` files)  
**Target Platform**: Linux/Windows server (local dev on Windows)  
**Project Type**: HTTP web service (FastAPI)  
**Performance Goals**: No change вЂ” normalisation is O(n) arithmetic on the existing price series; negligible overhead.  
**Constraints**: Cache stores raw sub-unit values; normalisation is applied at response time only (no cache invalidation).  
**Scale/Scope**: Single-service; impacts every response from `/securities/{ticker}/price` and `/securities/{ticker}/history` where the source reports a sub-unit currency.

## Design: SubUnitNormaliser

```python
# app/services/minor_unit.py

_MINOR_UNIT_MAP: dict[str, tuple[str, float]] = {
    "GBp": ("GBP", 100.0),
    "USd": ("USD", 100.0),
}

class SubUnitNormaliser:
    def is_minor_unit(self, currency: str) -> bool: ...
    def normalise(self, currency: str, price: float) -> tuple[str, float]: ...
    def normalise_series(
        self, currency: str, series: list[tuple[date, float]]
    ) -> tuple[str, list[tuple[date, float]]]: ...
```

Lookup uses `.lower()` on the third character for case-insensitive matching against the known map keys. Any currency not in the map passes through unchanged.
